# 🧪 Test Strategy – OpenCart Project

## 📌 Introduction
This document defines the overall approach and methodology for testing the OpenCart E-commerce application.

---

## 🎯 Test Objectives
- Ensure all functionalities work as expected
- Identify defects early
- Deliver a high-quality product

---

## 📍 Scope of Testing

### In-Scope:
- Login & Registration
- Product Search
- Add to Cart
- Checkout
- Payment Gateway
- Order Management

### Out-of-Scope:
- Third-party integrations (if not accessible)

---

## 🧪 Test Levels
- Unit Testing (by Developers)
- Integration Testing
- System Testing
- User Acceptance Testing (UAT)

---

## 🔍 Test Types
- Functional Testing
- Non-Functional Testing
- Regression Testing
- Smoke & Sanity Testing
- UI Testing

---

## 🧠 Test Techniques
- Black Box Testing
- Boundary Value Analysis
- Equivalence Partitioning
- Exploratory Testing
- Error Guessing

---

## ⚙️ Test Approach
- Start with Smoke Testing
- Perform Functional Testing using test cases
- Log defects in JIRA
- Perform Regression Testing after fixes
- Repeat test cycles until stable build

---

## 🌐 Test Environment
- Windows, macOS
- Chrome, Firefox, Edge, Safari
- Mobile devices (Android & iOS)

---

## 📦 Test Deliverables
- Test Plan
- Test Cases
- Test Execution Report
- Bug Reports

---

## 👥 Roles & Responsibilities
- Tester: Execute test cases, report defects
- Developer: Fix defects
- Test Lead: Manage testing process

---

## ⚠️ Risk & Mitigation
- Risk: Unstable build  
  Mitigation: Perform smoke testing  

- Risk: Time constraints  
  Mitigation: Prioritize critical scenarios  

---

## ✅ Exit Criteria
- All critical test cases executed
- Major defects resolved
- Test summary report prepared
