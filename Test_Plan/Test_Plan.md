# 🧪 Test Plan – OpenCart Application

## 📌 Objective
To ensure the e-commerce application meets all functional requirements, provides a user-friendly experience, and performs efficiently.

---

## 📍 Scope of Testing

### In-Scope:
- Login & Registration
- Add to Cart
- Checkout Process
- Payment Gateway
- Order Management

### Out of Scope:
- Third-party integrations (if not accessible)
- Admin panel testing (optional)

---

## 🧪 Testing Types
- Functional Testing
- Smoke Testing
- Sanity Testing
- Regression Testing
- UI Testing

---

## 🌐 Test Environment
- OS: Windows 10, macOS
- Browsers: Chrome, Firefox, Edge, Safari
- Mobile: Android (Chrome), iOS (Safari)
- Application URL: https://demo.opencart.com

---

## 🛠 Tools Used
- JIRA (Bug Tracking)
- Excel (Test Cases)
- Snipping Tool (Screenshots)

---

## 🐞 Defect Reporting Process
- Identify defect
- Log bug in JIRA
- Assign severity & priority
- Track till closure

---

## 📅 Test Execution Strategy
1. Perform Smoke Testing
2. Execute Test Cases
3. Log Defects
4. Perform Regression Testing
5. Prepare Test Summary Report

---

## ✅ Entry Criteria
- Requirements available
- Test cases prepared
- Test environment ready

## ✅ Exit Criteria
- All test cases executed
- Critical defects fixed
- Test report generated

---

## ⚠️ Risks & Mitigation
- Risk: Build not stable  
  Mitigation: Wait for stable build  

- Risk: Time constraints  
  Mitigation: Prioritize critical test cases
